# UI モーション素材依頼書（v0.22.0 motion-pass-1 連動）

> **作品**: 転生したらオセロ世界でした！（Reincarnated as an Othello Player）
> **連動コード**: `othello-game v0.22.0 · motion-pass-1`
> **作業内容**: 別 AI に画像生成を依頼するための素材仕様書
> **使い方**: 各依頼ブロックの `prompt` 内をそのままコピペして画像生成
>           AI（ChatGPT / Claude / Gemini / Midjourney 等）の入力欄に
>           貼ると、納品物が手に入る粒度で書いています。

---

## 全体像

コード側（v0.22.0）でモーション・エフェクトを placeholder CSS で先行
実装済み。下記 5 種の素材が届いたら CSS の `background: url(...)` を
差し替えるだけで完成形に進化します。素材が遅れても機能としては全部
動くので、優先度順に小出しでも OK。

| # | 名前 | 用途（コード上） | 優先度 | 形式 | 数量 |
|---|---|---|---|---|---|
| 1 | 墨絵風オーナメント仕切り | 各モーダル h2 装飾 | 🟡 中 | SVG | 5 種 |
| 2 | 桜花びらシルエット | 章クリア紙吹雪 | 🟢 余裕 | SVG | 3 種 |
| 3 | アプリアイコン更新 | favicon / PWA | 🔴 高 | SVG | 1 |
| 4 | Sumi-e 思考中ブラシ | AI 思考インジケーター | 🟢 余裕 | SVG | 4 frames |
| 5 | 和柄背景パターン | ステージ背景 watermark | 🟡 中 | PNG | 1 |

## 依頼 #1 — 墨絵風オーナメント仕切り（5 SVG）

```
【プロジェクト】和風オセロゲーム『転生したらオセロ世界でした！』

【用途】
タイトル画面、設定モーダル、ゲームオーバーモーダルの h2 タイトル上下
に置く装飾仕切り。現状はテキスト記号「· OPENING ·」だけで殺風景。
ユーザー視点で見える場所:
  - GameOver モーダルの「— Final Result —」上下
  - 設定モーダルの「Setup」「Story」セクション見出し上
  - リプレイヘルプモーダルのサブタイトル「— Replay Controls —」周辺

【ファイル形式】SVG（1 ファイル 1 仕切り、合計 5 種）
【ファイル名】public/ornaments/divider-{thin,bold,flourish,end,double}.svg
【ViewBox】0 0 240 24（横長）

【色指定】
- fill / stroke ともに「currentColor」固定（CSS 側で着色する）
- SVG 内には絶対色を入れない
- 線の不透明度のみ可（attribute opacity="0.6" 等）

【スタイル】墨筆の一筆書き。線の始点・終点に微かな滲み（bleed）

【5 バリエーション】
  thin     — 細い直線一本、両端が筆らしくテーパー
  bold     — 太めの直線。中央が一番太く両端で細る（呼吸する筆）
  flourish — 中央に小さな菱形 or 印章風シンボルが乗り、左右に薄い線
  end      — 右側が筆を上げるようにかすれて消える（章末用）
  double   — 上下平行の二本線、わずかに長さを変える

【参考】
- 江戸時代の版本の章扉
- 棟方志功の筆致
- Shippori Mincho の縦線

【避ける】
- 西洋的なフローリッシュ
- ベクター CG ぽい完全均一線
- グラデ・ドロップシャドウ・立体感・写実

【納品】5 ファイル、UTF-8、各 < 4KB
```

```

【3 バリエーション】

  petal-2 — やや傾いた角度、ねじれ感あり
  petal-3 — 半分丸まって落ちている動的なシルエット

- フォトリアル
```

## 依頼 #5 — 和柄背景パターン（1 PNG タイル）

```
【プロジェクト】同上

【用途】
タイトル画面とゲーム画面の背景に薄く敷く水印（watermark）。
既存の dark gradient の上に CSS の `background-repeat: repeat` で
重ねる。極めて控えめにすること。
ユーザー視点で見える場所:
  - タイトル画面背景（Stage 全体）
  - ゲーム画面背景（盤面の周辺）

【ファイル形式】PNG（透明背景、シームレスタイル）
【ファイル名】public/textures/wagara-tile.png
【サイズ】256×256 px、72dpi、< 30KB

【色指定】
- 薄い金色の線のみ #c9a961 alpha 8%
- 背景は完全透明（alpha 0）

【スタイル】以下のいずれか 1 種を選定して納品（3 種試作 → ユーザーが選ぶ）

  選択肢 A — 麻の葉文（asanoha）
    六角星状の幾何模様。江戸庶民の代表柄。
    凜とした幾何感が出る

  選択肢 B — 青海波（seigaiha）
    同心円の波形が連なる柄。流れ・継続の象徴。
    動きを感じさせる

  選択肢 C — 七宝（shippo）
    円が連結して花のような連続模様。仏教由来。
    柔らかく繊細

【シームレスタイル必須】
上下左右で違和感なく繰り返せること（256px の境界で線がぴたっと
継続する）

【避ける】
- グラデーション
- 影
- 写実
- 色彩鮮やか
- 商標
- 浮世絵調の人物・動物モチーフ

【納品】
- まず 3 種類のサンプル（各 256×256 PNG、低解像度プレビュー可）
- ユーザー選定後、最終 1 ファイル
```

## 依頼 #6 — 握り石コインフリップ素材（任意・3 SVG）

> v0.24.0 で「先攻／後攻」のランダム決定演出を実装済み（CSS のみで
> 完結する 3D コイン）。本依頼は **その上位互換**として、より和的な
> 「石を握る／開く」演出に差し替えるための素材。CSS 版で十分機能して
> いるので、依頼しなくても破綻しない。

```
【プロジェクト】和風オセロゲーム『転生したらオセロ世界でした！』

【用途】
ストーリーモード章開始時 + フリーモードの「ランダム」設定時に、先攻
（黒）／後攻（白）が決まる瞬間の演出。現状 CSS の 3D コインフリップ
で代替済み。本素材を入れ替えると、和の演出として「石を握る → 開く」
の動きに進化させられる。

【ファイル形式】SVG（モノクロシルエット、3 種）
【ファイル名】public/ornaments/grip-{closed,opening,open-black,open-white}.svg
【ViewBox】0 0 120 120

【色指定】
- fill / stroke ともに「currentColor」固定
- 中の石を表す部分だけは `fill="#0a0805"`（黒石）または
  `fill="#ebe2cc"`（白石）に固定可

【4 フレーム】
  closed       — 拳を握った状態。指で石を完全に隠す。
                  人差し指から小指まで揃った輪郭、親指は手前にきて
                  覆う形
  opening      — 拳が少し開いた状態。指の間から石の影がちらりと
                  見える。緊張感を演出
  open-black   — 完全に開いた手のひらに黒石が乗っている
  open-white   — 完全に開いた手のひらに白石が乗っている

【スタイル】
- 浮世絵 / 墨絵 風の手の輪郭。線は太め（2-3px 換算）
- 写実的すぎない、抑制された美しさ

【避ける】
- 5 本指の写実描画（簡略化された 4 指でも可）
- 影・グラデ・立体感
- 漫画的な誇張
- 西洋的な手の描き方

【納品】4 ファイル、各 < 4KB

【実装側との連携】
- `closed` → `opening` → `open-black|open-white` の 3 フレームを
  各 0.5s で順送り、最後のフレームだけ 1s 表示
- 黒石／白石どちらが選ばれるかはランタイム抽選
- src/App.tsx の `<FirstPlayerRoll>` コンポーネントを差し替え
```

## 推奨進行

1. **🔴 まず `依頼 #3` ファビコン**（PWA 印象の核、優先度最高）
2. **🟡 次に `依頼 #1` 仕切り** + **`依頼 #5` 背景パターン**
   （UI 品質を一段引き上げる）
3. **🟢 最後に `依頼 #2` 桜** + **`依頼 #4` Sumi-e 思考中**
   （演出の艶感を加える、無くても破綻しない）

各依頼は独立しているので並列発注可能。Midjourney 等で先に複数案を
並べてユーザーに選んでもらう運用が安全。
