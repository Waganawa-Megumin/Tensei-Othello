## 🎯 共通前提・準備

### クリーンスタート方法

1. ブラウザの開発者ツール → Application → Local Storage を開く
2. `tensei-othello-*` 関連の全キーを削除
3. ページをリロード
4. キャラ選択画面が初期状態 (PLR00 のみ選択可) であることを確認

### 確認すべき LocalStorage キー (Phase 3 で追加された想定)

| キー | 用途 | 期待される値の遷移 |
|---|---|---|
| `unlockedPlayers` | 解放済 PLR の配列 | `["PLR00"]` → `["PLR00","PLR01"]` |
| `unlockedOpponents` | 解放済 OPP の配列 (隠し系) | `[]` → `["OPP21","OPP22"]` |
| `trueEndingAchieved` | 真エンディング達成フラグ | `false` → `true` |
| `chapterProgress.PLR00` | PLR00 のクリア章 | 〜20 |
| `chapterProgress.PLR01` | PLR01 のクリア章 | 0 → 20 |

> ⚠️ キー名は実装を見て調整してください。Claude Code が違う命名を使っている可能性あり

## T-01: 通常エンディング達成 → PLR01 アンロック

### 目的
PLR00 (現代ハルキ) で章 20 をクリアし、PLR01 (英霊ハルキ) が解放されることを確認

### 前提
- クリーンインストール状態
- PLR00 のみがキャラ選択可能

### 手順
| # | 操作 | 期待動作 |
|---|---|---|
| 1 | キャラ選択画面で **PLR00** を選択 | PLR00 のアバター表示・名言「いざ、参る」表示 |
| 2 | ストーリーモードで章 1 〜 19 を順番にクリア | 各章クリア後、次章ロック解除 |
| 3 | 章 20 (フード姿ゼロ) に挑戦 | OPP20 アバター表示、名言「全ての変分は計算済み。詰みだ」表示 |
| 4 | **章 20 の最終一手 (満杯になる手)** を打つ | ⚠️ **凍結バグ確認ポイント** — 正常終了するか確認 |
| 5 | bossPost (PLR00 版) 表示 | フードが落ちる演出、フード無しゼロアバターに切替 |
| 6 | 通常エンディング演出 → タイトル画面に戻る | キャラ選択画面で **PLR01 が解放表示** |
| 7 | LocalStorage を確認 | `unlockedPlayers` に `"PLR01"` が追加 / `chapterProgress.PLR00 = 20` |

### 合格基準
- [ ] 章 20 で凍結しない
- [ ] PLR01 がキャラ選択画面に出現
- [ ] LocalStorage に解放状態が永続化されている
- [ ] OPP21/22 はまだ `???` (黒塗り) 表示のまま (= 真エンディング前)

### 既知の問題

## T-09: エッジケース集

### 9-1. PLR01 で章 20 に**敗北**した場合
- 通常通り敗北画面 → 章 20 リトライ可能
- `trueEndingAchieved` フラグは `false` のまま
- OPP21/22 は `???` のまま

### 9-2. T-04 (フルスクリーン演出) 中にブラウザリロード
- リロード後の状態は実装依存
- 推奨: 演出途中での中断は `trueEndingAchieved` を**まだ立てない**設計 (= 演出完了でフラグ立て)
- もしフラグが立つ実装なら、再ログインで OPP21/22 が解放されているはず → 演出だけ見逃した状態
- → **テスト実施: どちらの挙動か確認し仕様として記録**

### 9-3. T-05 のエピローグ中に「タイトルに戻る」を押す
- 推奨: ボタンを**無効化**するか、確認ダイアログを出す
- もし押せた場合 → 同上 (フラグの扱いを記録)

### 9-4. PLR00 で再度章 20 を打つ (真エンディング後)
- 通常通り PLR00 ルートの章 20 (フード姿ゼロ) と対戦できる
- bossPost は PLR00 版 (通常エンディング) のみ
- 既にアンロック済の OPP21/22 が**追加で解放されない** (重複処理がない)

### 9-5. PLR01 で章 20 を**もう一度**打つ (真エンディング後)
- 通常通り PLR01 ルートの章 20-B シネマティックが再生される
- 真エンディング演出は再生される (= 何度でも見られる) or スキップ可能
- → **どちらの仕様か確認**

### 9-6. OPP21 vs PLR01 のフリーモード対戦
- PLR01 (英霊ハルキ) で OPP21 (現世帰還ゼロ) と戦う
- 演出は通常のフリーモード対戦 (シネマティックは出ない)
- → 妙な台詞ループや矛盾が発生しないか確認

### 9-7. 章 20-B 挿絵の表示確認
- LS と PT の両方が `public/avatars/chapters/` に配置されているか
- 章 20-B 挿絵は **PLR01 ルートのみ** で表示され、PLR00 ルートでは表示されない

## 🔧 不具合発生時の確認順序

1. **コンソールエラー**を開発者ツールで確認
2. **LocalStorage の状態**をスナップショット取得
3. **i18n のキー**が日英で揃っているか確認 (片方が `undefined` ならフォールバック確認)
4. **アバター画像のパス**が正しいか確認 (404 の場合は manifest 確認)
5. **AI Worker のログ**を確認 (`console.log("worker request"…)` 等)

### 凍結バグ (T-01 ステップ 4) のデバッグ手順
- `engine` の純関数で当該盤面を再現するテストを書く
- AI Worker から最終手番のリクエスト/レスポンスをログ化
- ステートマシンの `WAITING_AI → GAME_OVER` 遷移が定義されているか確認
- 詳細は `BACKLOG.md` または別途デバッグセッションを参照
