# 🚦 状態遷移モデル (各 PLR が辿る 5 段階)

```
[STAGE 0] 未着手           ← PLR09-20
   ↓ ChatGPT 発注 (キャラ portrait + 専用背景)
[STAGE 1] キャラのみ受領    ← PLR00, PLR02-08 (現状)
   ↓ 背景受領 + 透過処理品質確認
[STAGE 2] 背景受領          ← (該当なし)
   ↓ icon 円形クロップ合成
[STAGE 3] 3 アセット揃い    ← (該当なし)
   ↓ 名言確定 + アンロック条件確定 + spec FINAL 化
[STAGE 4] FINAL             ← PLR01 のみ
```

**現状の偏在**: 8 体が STAGE 1 で滞留。2 → 3 → 4 へ進めるには **背景生成 → icon 合成 → spec FINAL 化** の 3 工程。

# 🎯 制作優先順位 (推奨案 A: 段階的完了)

`HANDOFF_v3_PLR_status.md` の推奨に沿って **PLR00 → PLR02-08 → PLR09-20** の三段制作を提案。

## Wave 1: PLR00 完全形化 (最優先)

**理由**: デフォルト主人公。毎回ゲーム起動時に表示される最重要アバター。

```
作業:
1. 既存キャラ画像 (旧 Claude flood-fill 処理) の品質再評価
   → 品質に問題があれば Adobe 再透過処理 (作者依頼)
2. 専用背景の発注 (現代日本の街角・通学路 / PLR01 遷移空間 / OPP21 帰還の街角の三部作起点)
3. icon 合成
4. spec.md FINAL 化 (PLR01 と並ぶ「お手本」レベルに)
```

成果物: `avatars-package/players/PLR00_default/{character,background,icon}.png + spec.md`

## Wave 2: PLR02-08 の完全形化 (7 体 = 21 ファイル)

**理由**: キャラ画像が既にあるため、**背景 + icon + spec FINAL 化** だけで完結。最も効率が良い。

## Wave 3: PLR09-20 の新規制作 (12 体 = 48 ファイル)

**理由**: 最も労力大。`reference_originals/PLRxx_*.png` (512×512) を ChatGPT 参照画像として使い、**新フォーマット 1024×1024** で再生成。

各キャラの背景案: 各 spec.md の「背景の世界観」セクションに記載済 (案段階)。作者承認待ち。

工程: 1 体あたり ChatGPT 発注 (キャラ + 背景) → Adobe 透過処理 (キャラのみ) → 受領処理 → spec FINAL 化
